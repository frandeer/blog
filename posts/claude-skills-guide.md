# Claude Skills: AI 에이전트 시대의 새로운 패러다임

2025년, AI는 단순히 질문에 답하는 도구를 넘어 실제 업무를 수행하는 '에이전트'로 진화하고 있습니다. Claude의 Skills는 이러한 변화의 핵심에 있습니다. 이 글에서는 Claude의 에이전트 생태계를 구성하는 빌딩 블록들을 살펴보고, 실무에서 어떻게 활용할 수 있는지 알아보겠습니다.

## AI 에이전트의 핵심: 점진적 공개(Progressive Disclosure)

Claude Skills의 가장 혁신적인 개념은 **점진적 공개(Progressive Disclosure)**입니다. 이는 마치 잘 정리된 매뉴얼처럼 작동합니다:

1. **메타데이터 (~100 토큰)**: 먼저 목차만 확인
2. **전체 지시사항 (<5k 토큰)**: 필요하면 해당 챕터 읽기
3. **번들 파일/스크립트**: 정말 필요할 때만 상세 부록 참조

이 방식의 핵심은 **컨텍스트 효율성**입니다. 모든 정보를 한 번에 로드하는 대신, 필요한 시점에 필요한 만큼만 가져옵니다.

```markdown
# 예시: 브랜드 가이드라인 Skill

## 메타데이터 (항상 로드)
name: brand-guidelines
description: 우리 회사의 브랜드 색상, 타이포그래피, 레이아웃 규칙

## 지시사항 (관련 작업 시 로드)
- 주 색상: #1E3A8A (네이비 블루)
- 보조 색상: #10B981 (에메랄드 그린)
- 폰트: Pretendard Variable

## 에셋 파일 (실제 사용 시 로드)
- logo.svg
- color-palette.json
- typography-guide.pdf
```

## 5가지 빌딩 블록: 언제 무엇을 써야 할까?

### 1. Skills: 재사용 가능한 전문 지식

**핵심 개념**: 특정 도메인의 전문성을 패키지화

**언제 사용하나요?**
- ✅ 같은 지시를 반복적으로 입력하고 있을 때
- ✅ 조직 전체가 공유해야 하는 절차가 있을 때
- ✅ 코드, 스크립트, 참조 파일이 함께 필요할 때

**실무 예시**:

```markdown
# API 보안 리뷰 Skill

## 검토 절차
1. OWASP Top 10 기준으로 취약점 검사
2. 인증/인가 로직 검증
3. SQL Injection, XSS 가능성 확인
4. 민감 데이터 노출 여부 확인

## 출력 형식
- 심각도: Critical/High/Medium/Low
- 위치: 파일명:라인번호
- 설명: 왜 문제인지, 어떻게 악용될 수 있는지
- 해결 방법: 구체적인 코드 예시
```

**왜 강력한가?**: 한 번 만들면 모든 대화, 모든 프로젝트에서 재사용 가능합니다.

### 2. Prompts: 순간의 지시

**핵심 개념**: 일회성, 대화형 요청

**언제 사용하나요?**
- ✅ 한 번만 필요한 작업
- ✅ 즉각적인 컨텍스트 제공
- ✅ 대화를 통한 점진적 개선

**실무 예시**:

```
"이 데이터를 분석해서 트렌드를 찾아줘"
"좀 더 전문적인 톤으로 바꿔줘"
"이 코드를 TypeScript로 변환해줘"
```

**Prompt → Skill 전환 시점**: 같은 프롬프트를 3번 이상 사용한다면 Skill로 만들 때입니다.

### 3. Projects: 지속적인 컨텍스트

**핵심 개념**: 특정 이니셔티브를 위한 독립적인 작업 공간

**언제 사용하나요?**
- ✅ 배경 지식이 모든 대화에 필요할 때
- ✅ 팀 협업이 필요할 때
- ✅ 문서, 데이터를 중앙화하고 싶을 때

**실무 예시**:

```
프로젝트: "Q4 제품 출시"

업로드된 지식:
- 시장 조사 보고서 (50페이지)
- 경쟁사 분석 (30페이지)
- 제품 스펙 문서 (100페이지)
- 고객 피드백 데이터

프로젝트 지시사항:
"모든 분석은 우리 제품 차별화 관점에서 진행하고,
구체적인 증거와 실행 가능한 제안을 포함하세요."
```

**프로젝트 vs Skill**:
- **Projects**: "이것을 알아야 해" (정적 지식)
- **Skills**: "이렇게 해야 해" (동적 절차)

### 4. Subagents: 특수 작업 전문가

**핵심 개념**: 독립적인 컨텍스트와 권한을 가진 전문 에이전트

**언제 사용하나요?**
- ✅ 복잡한 워크플로우를 여러 단계로 분리할 때
- ✅ 특정 도구만 사용하도록 제한해야 할 때
- ✅ 병렬 처리가 필요할 때

**실무 예시**:

```yaml
# code-reviewer 서브에이전트
name: code-reviewer
description: 코드 품질 및 보안 리뷰 전문
tools: Read, Grep, Glob  # Write, Edit 권한 없음!

시스템 프롬프트:
당신은 시니어 코드 리뷰어입니다.
1. 코드 품질 체크 (가독성, 유지보수성)
2. 보안 취약점 검사
3. 베스트 프랙티스 준수 여부
4. 테스트 커버리지 확인
```

```yaml
# test-generator 서브에이전트
name: test-generator
description: 단위 테스트 자동 생성
tools: Read, Write, Bash

시스템 프롬프트:
테스트 커버리지 80% 이상을 목표로
Jest/Vitest를 사용한 단위 테스트를 작성합니다.
```

**Subagent vs Skill**:
- **Skills**: 어떤 에이전트든 사용 가능한 전문성
- **Subagents**: 특정 작업을 위한 독립 에이전트

### 5. MCP: 외부 세계와의 연결

**핵심 개념**: AI와 실제 데이터/도구를 연결하는 표준 프로토콜

**언제 사용하나요?**
- ✅ 외부 데이터에 접근해야 할 때
- ✅ 비즈니스 도구와 통합해야 할 때
- ✅ 실시간 데이터가 필요할 때

**실무 예시**:

```javascript
// MCP 서버 예시: 회사 Google Drive 연결
{
  "mcpServers": {
    "google-drive": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-gdrive"],
      "env": {
        "GOOGLE_DRIVE_CREDENTIALS": "..."
      }
    },
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"]
    }
  }
}
```

**MCP vs Skill**:
- **MCP**: 데이터에 **접근**
- **Skill**: 데이터를 **어떻게 사용**할지

둘을 함께 사용하세요! MCP로 데이터를 가져오고, Skill로 처리 방법을 정의합니다.

## 실전 활용: 경쟁사 분석 에이전트 구축

이제 모든 빌딩 블록을 조합하여 실제 워크플로우를 만들어봅시다.

### 시나리오
스타트업 PM이 경쟁사의 AI 기능을 분석하고 우리가 공략할 수 있는 틈새 시장을 찾아야 합니다.

### 1단계: Project 설정

```
프로젝트명: "경쟁 인텔리전스"

업로드 자료:
- 산업 리포트 (Gartner, Forrester)
- 경쟁사 제품 문서
- 고객 피드백 (CRM 데이터)
- 이전 분석 리포트

프로젝트 지시사항:
우리 제품 전략 관점에서 경쟁사를 분석하세요.
차별화 기회와 신흥 트렌드에 집중하고,
구체적 증거와 실행 가능한 제안을 제시하세요.
```

### 2단계: MCP로 데이터 소스 연결

```json
{
  "google-drive": "연결 ✅ (공유 리서치 문서)",
  "github": "연결 ✅ (경쟁사 오픈소스 저장소)",
  "web-search": "연결 ✅ (실시간 시장 정보)"
}
```

### 3단계: Skill 생성

```markdown
# competitive-analysis Skill

## 분석 프레임워크

### 1. 시장 포지셔닝 분석
- 타겟 고객층
- 가격 전략
- 주요 메시징

### 2. 기능 비교 매트릭스
| 기능 | 우리 | 경쟁사A | 경쟁사B |
|------|------|---------|---------|
| AI 자동화 | ✅ | ✅ | ❌ |
| 엔터프라이즈 보안 | ✅ | ❌ | ✅ |

### 3. 틈새 시장 식별
- 경쟁사가 놓친 시장
- 고객 불만족 영역
- 신흥 트렌드 기회

### 4. 실행 제안
- 우선순위 (High/Medium/Low)
- 예상 임팩트
- 필요 리소스
```

### 4단계: Subagent 구성

```yaml
# market-researcher
name: market-researcher
description: 시장 트렌드 및 경쟁 환경 리서치
tools: Read, Grep, WebSearch

프롬프트:
시장 조사 전문 분석가로서:
1. 권위 있는 소스 우선 (Gartner, Forrester)
2. 정량 데이터 수집 (시장 점유율, 성장률)
3. 정성 인사이트 분석 (애널리스트 의견)
4. 트렌드 패턴 도출
출처와 신뢰도 명시 필수.
```

```yaml
# technical-analyst
name: technical-analyst
description: 기술 아키텍처 및 구현 분석
tools: Read, Bash, Grep

프롬프트:
기술 아키텍트로서 경쟁사 기술 분석:
1. 공개 저장소 및 기술 문서 검토
2. 아키텍처 패턴 및 기술 스택 평가
3. 확장성/성능 접근 방식 분석
4. 기술적 강점과 한계 파악
우리 제품 결정에 도움되는 실행 가능한 인사이트 제공.
```

### 5단계: 에이전트 실행

**사용자**: "상위 3개 경쟁사가 새로운 AI 기능을 어떻게 포지셔닝하고 있는지 분석하고, 우리가 공략할 수 있는 틈새를 찾아줘"

**내부 작동 순서**:

```
1. 📁 Project 컨텍스트 로드
   → 업로드된 리서치 문서 활성화
   → 프로젝트 지시사항 적용

2. 🔌 MCP 연결 활성화
   → Google Drive에서 최근 경쟁사 브리프 검색
   → GitHub에서 기술 데이터 수집

3. 🎯 Skills 적용
   → competitive-analysis 프레임워크 로드
   → 분석 구조 제공

4. 🤖 Subagents 실행 (병렬)
   → market-researcher: 산업 데이터 수집
   → technical-analyst: 기술 구현 검토

5. 💬 Prompt로 세부 조정
   "특히 헬스케어 엔터프라이즈 고객에 집중해줘"
```

**결과물**: 여러 데이터 소스를 활용하고, 분석 프레임워크를 따르며, 전문화된 에이전트의 인사이트를 결합한 종합 경쟁 분석 보고서

## 한국 개발자를 위한 실무 팁

### 1. 팀 온보딩 자동화

```markdown
# onboarding-guide Skill

## 개발 환경 설정
1. Node.js 18+ 설치
2. pnpm 설치: `npm i -g pnpm`
3. 저장소 클론: `git clone ...`

## 코드 컨벤션
- ESLint + Prettier 사용
- 커밋 메시지: `feat:`, `fix:`, `docs:` 형식
- PR 전 `pnpm test` 필수

## 아키텍처 개요
- `/src/api`: REST API 엔드포인트
- `/src/services`: 비즈니스 로직
- `/src/utils`: 공통 유틸리티
```

### 2. 코드 리뷰 자동화

```yaml
# Subagent: korean-code-reviewer
name: korean-code-reviewer
description: 한국어로 친절하게 설명하는 코드 리뷰어
tools: Read, Grep

시스템 프롬프트:
시니어 개발자로서 코드를 리뷰합니다.
- 한국어로 친절하게 설명
- 왜 문제인지 이유 설명
- 개선 방법 제시
- 칭찬할 점도 언급

출력 형식:
👍 잘한 점
⚠️ 개선 필요
💡 제안사항
```

### 3. 기술 문서 작성

```markdown
# tech-writing Skill

## API 문서 템플릿

### 엔드포인트
`POST /api/users`

### 설명
새로운 사용자를 생성합니다.

### 요청
\`\`\`json
{
  "email": "user@example.com",
  "name": "홍길동"
}
\`\`\`

### 응답
\`\`\`json
{
  "id": "user_123",
  "email": "user@example.com",
  "createdAt": "2024-11-18T12:00:00Z"
}
\`\`\`

### 에러 처리
- 400: 잘못된 요청
- 409: 이메일 중복
- 500: 서버 오류
```

## 선택 가이드: 빠른 참조표

| 상황 | 사용할 것 | 이유 |
|------|-----------|------|
| 같은 지시를 계속 반복 | **Skill** | 재사용 가능, 컨텍스트 절약 |
| 한 번만 필요한 작업 | **Prompt** | 빠르고 유연함 |
| 프로젝트 전반의 배경 지식 | **Project** | 지속적 컨텍스트 |
| 독립적 작업 분리 | **Subagent** | 권한 제어, 병렬 처리 |
| 외부 데이터 필요 | **MCP** | 실시간 연결 |
| 절차 + 데이터 둘 다 | **Skill + MCP** | 접근 + 사용법 |
| 팀 전체 공유 필요 | **Skill** | 조직 전체 재사용 |
| 일시적 협업 | **Project** | 팀 작업 공간 |

## 실무 도입 로드맵

### Phase 1: 개인 생산성 (1주차)
1. 자주 하는 작업을 Skill로 만들기
2. Project로 작업 정리하기
3. Prompt 최적화

### Phase 2: 팀 협업 (2-4주차)
1. 팀 공통 Skill 라이브러리 구축
2. 프로젝트 템플릿 생성
3. MCP로 Slack, GitHub 연결

### Phase 3: 워크플로우 자동화 (1-2개월)
1. Subagent로 복잡한 워크플로우 분해
2. CI/CD 통합
3. 모니터링 및 최적화

## 마치며: AI 에이전트 시대의 개발자

Claude Skills는 단순한 기능이 아니라 **새로운 개발 패러다임**입니다.

이제 우리는:
- 코드를 **작성하는** 대신 **설명**합니다
- 반복 작업을 **수행하는** 대신 **자동화**합니다
- 지식을 **검색하는** 대신 **컨텍스트로 제공**합니다

핵심은 **적재적소에 올바른 도구를 사용**하는 것입니다:

```
데이터 접근 (MCP)
  → 배경 지식 (Project)
    → 전문성 (Skill)
      → 작업 실행 (Subagent)
        → 실시간 조정 (Prompt)
```

2025년은 AI 에이전트가 실제 업무를 수행하는 원년입니다. Skills와 함께 이 변화의 최전선에 서보세요.
